## 4. 数据库初始化

### 4.1 创建数据库

```bash
# 登录 PostgreSQL
psql -U postgres

# 在 psql 内执行
CREATE DATABASE birthday_db;
CREATE USER birthday_user WITH ENCRYPTED PASSWORD '你的密码';
GRANT ALL PRIVILEGES ON DATABASE birthday_db TO birthday_user;
\q
```

### 4.2 推送 Schema

项目使用 **Drizzle ORM** 管理数据库结构，无需手写 SQL。

```bash
# 设置数据库连接（临时，正式请写入 .env）
export DATABASE_URL="postgresql://birthday_user:你的密码@localhost:5432/birthday_db"

# 推送所有表结构
pnpm --filter @workspace/db run db:push
```

> 首次部署后，后端启动时会自动插入默认配置记录（settings 表）。

## 5. 环境变量配置

在项目根目录创建 `.env` 文件（**不要提交到 Git**）：

```env
# ── 数据库 ────────────────────────────────────────────────
DATABASE_URL=postgresql://birthday_user:你的密码@localhost:5432/birthday_db

# ── 后端服务 ─────────────────────────────────────────────
PORT=8080                          # 后端监听端口（Nginx 代理此端口）
NODE_ENV=production
SESSION_SECRET=替换为随机长字符串    # 至少 32 位随机字符
LOG_LEVEL=info                     # debug | info | warn | error

# ── 管理员 ───────────────────────────────────────────────
ADMIN_KEY=birthday-admin-2024      # 访问管理后台的密钥，建议修改

# ── DeepSeek AI（生日历史事件功能）───────────────────────
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxx   # 在 DeepSeek 平台申请

# ── 邮件提醒（QQ 邮箱）───────────────────────────────────
# 以下两项也可在管理后台"邮件配置"中设置，优先使用数据库配置
QQ_EMAIL=991067346@qq.com
QQ_EMAIL_AUTH=xxxxxxxxxxxxxxxx     # QQ 邮箱授权码（非登录密码）

# ── 微信公众号登录（可选）────────────────────────────────
WECHAT_APPID=wxxxxxxxxxxxxxxxxxx
WECHAT_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 前端构建变量

前端在构建时通过 `BASE_PATH` 环境变量指定挂载路径：

```bash
# 前端挂载在根路径（推荐）
export BASE_PATH=/

# 前端监听端口（构建预览用，生产由 Nginx 提供静态文件）
export PORT=3000
```

## 6. 构建与启动

### 6.1 构建前端静态文件

```bash
cd artifacts/birthday-app
BASE_PATH=/ PORT=3000 pnpm run build
# 产物位于 artifacts/birthday-app/dist/public/
```

### 6.2 构建后端

```bash
cd artifacts/api-server
pnpm run build
# 产物位于 artifacts/api-server/dist/index.mjs
```

### 6.3 启动后端服务

```bash
cd artifacts/api-server

# 加载环境变量并启动
export $(cat ../../.env | grep -v '^#' | xargs)
pnpm run start
# 或直接：node --enable-source-maps ./dist/index.mjs
```

后端启动成功后输出：

```
[INFO] Server listening {"port":8080}
[INFO] Scheduling daily birthday reminders
[INFO] Next birthday reminder run scheduled {"nextRun":"..."}
```

### 6.4 使用 PM2 管理进程（推荐生产使用）

```bash
npm i -g pm2

# 启动后端
pm2 start "node --enable-source-maps /path/to/artifacts/api-server/dist/index.mjs" \
  --name birthday-api \
  --env production

# 开机自启
pm2 save
pm2 startup
```

## 7. Nginx 反向代理配置

前端编译产物为纯静态文件，由 Nginx 直接托管；API 请求代理到后端。

```nginx
server {
    listen 80;
    server_name your-domain.com;   # 替换为你的域名或 IP

    # 前端静态文件
    root /path/to/artifacts/birthday-app/dist/public;
    index index.html;

    # 用户上传头像文件
    location /api/uploads/ {
        alias /path/to/artifacts/api-server/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # API 请求代理到后端
    location /api/ {
        proxy_pass         http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;

        # 文件上传大小限制（头像）
        client_max_body_size 15m;
    }

    # 前端 SPA：所有非 API 路由回退到 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

**HTTPS 配置（推荐，微信功能必须）：**

```bash
# 使用 Certbot 申请免费 SSL 证书
apt install certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
```

## 8. 微信公众号对接（可选）

> 开启后用户可通过微信 OAuth 登录，并接收生日提醒推送。

### 8.1 公众号后台配置

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「设置 → 公众号设置 → 功能设置」
3. 填写"网页授权域名"：`your-domain.com`（无需 https://）
4. 进入「开发 → 基本配置」复制 AppID 和 AppSecret

### 8.2 填写环境变量

```env
WECHAT_APPID=wxxxxxxxxxxxxxxxxxx
WECHAT_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 8.3 在管理后台启用

1. 访问 `https://your-domain.com` → 输入管理员密钥进入后台
2. 选择「微信配置」选项卡
3. 填写 AppID、AppSecret，将登录模式切换为"微信 OAuth"
4. 开启"消息通知"，配置公众号名称

### 8.4 OAuth 回调地址

无需手动配置，系统自动计算回调地址：

```
https://your-domain.com/api/auth/wechat/oauth/callback
```

## 9. 小程序 WebView 接入（可选）

生日通 H5 可嵌入微信小程序的 `<web-view>` 组件。

### 小程序端代码示例

```javascript
// pages/birthday/index.js
Page({
  onLoad() {
    // 构造 H5 页面 URL，附带小程序来源标识
    const url = encodeURIComponent('https://your-domain.com/?from=miniprogram');
    wx.navigateTo({
      url: `/pages/webview/index?url=${url}`
    });
  }
});
```

```xml
<!-- pages/webview/index.wxml -->
<web-view src="{{url}}" bindmessage="handleMessage"></web-view>
```

### 透传登录 Token（高级）

小程序可在 URL 中携带 `mp_token` 参数，H5 页面将自动读取并免登录跳转：

```javascript
const mpToken = 'your_generated_token';   // 由后端签发
const url = `https://your-domain.com/login?mp_token=${mpToken}`;
```

### 小程序后台配置

在微信小程序后台「开发 → 开发管理 → 开发设置」中：
- 添加业务域名：`your-domain.com`
- 添加 webview 域名：`your-domain.com`

## 10. 邮件提醒配置

系统使用 QQ 邮箱 SMTP 发送生日提醒，**推荐在管理后台配置**（优先级高于环境变量）。

### SMTP 参数

| 参数 | 值 |
|------|---|
| SMTP 服务器 | smtp.qq.com |
| 端口 | 465 |
| 加密 | SSL/TLS（开启） |
| 发件人邮箱 | 你的QQ邮箱 |
| 授权码 | QQ邮箱 → 设置 → 账户 → 开启SMTP服务获取 |

### 在管理后台配置

1. 访问管理后台 → 选择「邮件配置」选项卡
2. 填写上述 SMTP 信息
3. 点击"测试发送"验证配置
4. 设置提前提醒天数（如：1天前、3天前、7天前）
5. 设置每日发送时间（默认 08:00）

### 定时任务说明

系统在后端启动时自动注册定时任务：

- **生日邮件提醒**：每天 08:00 UTC+8 检查并发送提醒邮件
- **微信通知**：每天 08:00 UTC+8 向关注公众号的用户推送通知

## 11. 管理后台使用

访问地址：在域名后拼接 `/admin` 路径。

- 本地开发：`http://localhost:3000/admin`
- 正式环境：`https://your-domain.com/admin`

打开后输入管理员密钥即可进入。

> 管理员密钥默认为 `birthday-admin-2024`，**强烈建议在 `.env` 中修改 `ADMIN_KEY`**。

### 功能选项卡

| 选项卡 | 功能 |
|--------|------|
| 用户管理 | 查看所有注册用户、删除账号 |
| 微信配置 | 填写 AppID/Secret、切换登录模式、配置推送通知 |
| AI 模型 | 配置 DeepSeek API Key 和生成参数 |
| 消息通知 | 配置微信模板消息通知 |
| 邮件配置 | SMTP 参数、发送时机、测试邮件 |
| 内容配置 | 用户协议、隐私政策文本 |

## 12. 日志 & 监控

后端使用 **pino** 结构化日志，格式为 JSON。

```bash
# 实时查看日志（pm2 管理时）
pm2 logs birthday-api

# 格式化输出（需安装 pino-pretty）
pm2 logs birthday-api | pino-pretty
```

### 日志级别

| 级别 | 说明 |
|------|------|
| debug | 详细调试信息 |
| info | 正常请求记录（默认） |
| warn | 非致命异常 |
| error | 错误和服务中断 |

通过 `LOG_LEVEL=debug` 环境变量调整。

## 13. 常见问题

### Q: 前端页面空白 / 502 Bad Gateway

1. 确认后端进程正在运行：`pm2 status` 或 `curl http://127.0.0.1:8080/api/auth/me`
2. 检查 Nginx 配置语法：`nginx -t`
3. 查看后端日志：`pm2 logs birthday-api --lines 50`

### Q: 数据库连接失败

```bash
# 测试连接
psql "$DATABASE_URL" -c "SELECT 1"
```

确认 PostgreSQL 服务正在运行：`systemctl status postgresql`

### Q: 上传头像后图片不显示

- 确认 `artifacts/api-server/uploads/` 目录存在且有写入权限：
  ```bash
  mkdir -p /path/to/artifacts/api-server/uploads
  chmod 755 /path/to/artifacts/api-server/uploads
  ```
- 确认 Nginx `location /api/uploads/` 的 `alias` 路径正确

### Q: 邮件发送失败

- 确认已在 QQ 邮箱后台开启 **SMTP 服务** 并获取授权码（非 QQ 密码）
- 检查服务器是否封锁了 465 端口：`telnet smtp.qq.com 465`
- 在管理后台使用"测试发送"功能查看详细错误

### Q: 微信登录跳转失败

- 确认域名已在微信公众号后台添加为"网页授权域名"
- 确认回调地址使用 HTTPS
- 检查 AppID / AppSecret 是否正确填写

### Q: DeepSeek AI 历史事件不生成

- 确认 `DEEPSEEK_API_KEY` 有效且账户有余额
- 在联系人详情页点击"重新生成"按钮
- 查看后端日志中 `deepseek` 相关错误信息
